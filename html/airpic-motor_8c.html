void motor_setPWM ( unsigned int  motor,
unsigned int  delay,
unsigned int  duty 
)

Set the specific PWM waveform to be delivered to the motor.

Set the PWM output for a motor to have the specified delay time (delay being the % of the cycle before the signal goes high) and the specified duty cycle (% of the cycle that the signal is high). Setting the duty variable to 4096 is effectively the same as the motor_stop() function.

Be careful! Unpredictable behavior can occur if (delay + duty) > 4096. Any problems like this can be prevented by simply keeping delay at zero, or another very low number.

Parameters
motorThe motor to be written to. Acceptable range: 0-15.
delayDelay before the signal goes high. Acceptable range: 0-4095.
dutyDuty cycle of the signal. Duty cycle will be equal to (duty / 4095). Acceptable range: 0-(4095 - delay).

◆ motor_stop()

void motor_stop ( unsigned int  motor)

Stop the specified motor.

Force the specified motor to be fully off by setting the LEDx_OFF_H[4] bit. More about how this works can be read on page 23-24 of the PCA9685 datasheet. Writing a duty cycle to the motor motor_write() or motor_setPWM() will start the motor again.

Parameters
motorThe motor you wish to stop.
