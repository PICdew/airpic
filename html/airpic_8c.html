void airpic_config ( void  )

Configure the microcontroller for the Airpic library.

Running airpic_config will do the following:

  • Set the microcontroller RC clock divider to 1:1 (16MHz, assuming a PIC24F device using the internal oscillator)
  • Set all pins to be digital, by disconnecting them from the Analog-to -Digital Converter.
  • Set all RAx pins to be digital outputs.
  • Set RB0, RB1, RB11, and RB12 to be digital inputs. All other RBx pins are digital outputs.
  • Set the UART1 peripheral's TX pin to pin RP10
  • Set the UART1 peripheral's RX pin to pin RP11

void airpic_debugger_enable ( unsigned int  baudSelect)

Enable the debugger serial port.

Messages are sent in the standard TTL format (RS-232, but with 0 and 3.3V logic levels).

If the debugger port is already enabled, the baudrate will not be changed. The baudrate of the port can only be changed if it has been disabled.

Parameters
baudSelectThe value written to U2BRG. The DEBUGGER_BAUDRATE_xxx macros are a convenient way of setting the UART to common baudrates.

void airpic_debugger_printnum ( unsigned int  val,
unsigned short  format

Print a single value (up to 16 bits in length) in the given format.

Parameters
valThe 16-bit value to be printed.
formatDescribes the way 'val' is printed. Use the macros BIN, DEC, or HEX to print the 'val' in binary, decimal, or hex respectively. If the value of 'format' is not one of the recognized values, 'val' will be printed as a single ASCII character. Binary and hex formats both will show leading zeroes; decimal format does not show leading zeroes.
